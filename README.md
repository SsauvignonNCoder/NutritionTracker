# Грузинская кухня · Питание

Справочник: рецепты, план питания на неделю, список покупок. Без логина — статичный сайт, читать с телефона (в т.ч. как Telegram Mini App).

Список покупок считается **автоматически** из ингредиентов выбранных на неделю рецептов. Блюда в плане можно заменять прямо в интерфейсе — замены сохраняются навсегда через Supabase.

---

## Запуск с нуля

Ниже — полный путь от пустой папки до рабочего сайта по ссылке. Если что-то из этого уже есть (Node.js, GitHub-аккаунт, аккаунт Vercel) — просто пропусти этот шаг.

### Шаг 0. Что понадобится

- Аккаунт на [GitHub](https://github.com) — бесплатный
- Аккаунт на [Vercel](https://vercel.com) — бесплатный, удобно входить через GitHub
- Аккаунт на [Supabase](https://supabase.com) — бесплатный, нужен только для того, чтобы замены блюд в плане сохранялись навсегда. Без этого шага сайт работает полностью, просто замена блюда сбросится при следующем визите.
- Node.js на компьютере — нужен **только** если хочешь проверять сайт локально перед заливкой. Если такой привычки нет — можно пропустить шаги 1–2 и сразу перейти к шагу 3 (Vercel соберёт сайт сам).

Скачать Node.js: [nodejs.org](https://nodejs.org) → версия LTS → установить как обычную программу.

### Шаг 1. Распаковать проект

Распакуй архив `nutrition-app.zip` в любую папку на компьютере, например на рабочий стол.

### Шаг 2. Завести проект в Supabase (чтобы замены блюд сохранялись)

Это отдельный, новый проект Supabase — никак не связанный с трекером тренировок.

1. Зайди на [supabase.com](https://supabase.com), войди (можно через GitHub)
2. **New Project** → дай название, например `nutrition-app` → выбери регион (любой, ближе к себе) → задай пароль базы (можно сгенерировать и просто сохранить — он почти никогда не понадобится) → **Create new project**. Подождать минуту, пока проект создастся.
3. В левом меню → **SQL Editor** → **New query**
4. Открой файл `supabase/schema.sql` из архива, скопируй всё содержимое, вставь в редактор → **Run**. Это создаст таблицу для хранения замен блюд.
5. В левом меню → **Settings** (шестерёнка) → **API**
6. Скопируй два значения:
   - **Project URL** (выглядит как `https://xxxxx.supabase.co`)
   - **anon public** ключ (длинная строка под "Project API keys")

Эти два значения понадобятся в шаге 4 при подключении к Vercel.

### Шаг 3. (Опционально) Проверить локально перед заливкой

Если хочешь сначала посмотреть на сайт у себя на компьютере:

1. Открой терминал (на Mac — приложение «Терминал», на Windows — `cmd` или PowerShell)
2. Перейди в папку проекта:
   ```
   cd путь/к/папке/nutrition-app
   ```
3. Скопируй `.env.example` в `.env` и впиши туда значения из Supabase (Project URL и anon key) из шага 2
4. Установи зависимости (один раз, займёт минуту):
   ```
   npm install
   ```
5. Запусти локальный сервер:
   ```
   npm run dev
   ```
6. Терминал покажет адрес вида `http://localhost:5173` — открой его в браузере.
7. Чтобы остановить — `Ctrl + C` в терминале.

Если просто хочешь сразу задеплоить без проверки — пропускай этот шаг.

### Шаг 4. Завести репозиторий на GitHub

1. Зайди на [github.com/new](https://github.com/new)
2. Назови репозиторий, например `nutrition-app`
3. Оставь его **Public** или **Private** — на твой вкус
4. Не добавляй README/.gitignore при создании — они уже есть в архиве
5. Нажми «Create repository»

Дальше залей файлы — самый простой способ без терминала:

1. На странице нового (пустого) репозитория нажми **«uploading an existing file»**
2. Перетащи туда **все файлы и папки** из распакованного `nutrition-app` (включая `src` целиком, `supabase`, `package.json`, `index.html`, `vite.config.js`, `.gitignore`, `.env.example`, `README.md`). **`.env` (без `.example`) заливать не нужно** — он и не должен попасть в репозиторий, .gitignore это исключает.
3. Внизу нажми **«Commit changes»**

Если привычнее через терминал:
```
cd путь/к/папке/nutrition-app
git init
git add .
git commit -m "Первая версия"
git branch -M main
git remote add origin https://github.com/ТВОЙ-АККАУНТ/nutrition-app.git
git push -u origin main
```

### Шаг 5. Подключить к Vercel

1. Зайди на [vercel.com](https://vercel.com), войди через GitHub
2. Нажми **«Add New...» → «Project»**
3. Найди в списке только что созданный репозиторий → **«Import»**
4. Vercel сам распознает, что это Vite-проект (Framework Preset: **Vite**) — ничего менять не нужно
5. **Перед тем как нажать Deploy**, разверни **«Environment Variables»** и добавь два значения из Supabase (шаг 2):
   - `VITE_SUPABASE_URL` = Project URL
   - `VITE_SUPABASE_ANON_KEY` = anon public ключ
   - Важно: для каждой переменной отметь и **Production**, и **Preview** — иначе сохранение замен блюд не будет работать на превью-деплоях
6. Нажми **«Deploy»**
7. Через 30–60 секунд получишь рабочую ссылку вида `nutrition-app-xxxx.vercel.app`

Готово — сайт работает, открывается с телефона и компьютера, замены блюд в плане сохраняются навсегда.

Если пропустил Supabase (шаг 2) — сайт всё равно полностью работает, просто замены блюд в плане будут сбрасываться при перезагрузке страницы. Подключить Supabase можно и позже: добавь переменные окружения в Vercel (Settings → Environment Variables этого же проекта) и сделай **Redeploy**.

### Шаг 6. (Опционально) Закрепить красивый домен

В настройках проекта на Vercel (**Settings → Domains**) можно либо оставить автоматический адрес, либо переименовать проект (тогда ссылка станет `nutrition-app.vercel.app` без случайных цифр, если имя свободно) — зайди в **Settings → General → Project Name**.

### Шаг 7. (Опционально) Сделать Telegram Mini App

Если хочешь открывать сайт прямо из Telegram-бота:

1. В Telegram напиши [@BotFather](https://t.me/BotFather)
2. Если бота ещё нет — `/newbot`, если есть — выбери существующего
3. Команда `/mybots` → выбрать бота → **Bot Settings → Menu Button → Configure Menu Button**
4. Вставь ссылку на сайт с Vercel (именно **постоянный** домен, не превью-ссылку)
5. Укажи название кнопки, например «Меню питания»

Готово — кнопка с сайтом появится в чате с ботом.

### Как обновлять сайт после первого запуска

1. Меняешь файл (`src/data/recipes.js` или `src/data/weeks.js`) — сам или присылаешь мне, что добавить, я меняю файл и отдаю обратно
2. Заливаешь изменённый файл на GitHub (через веб-интерфейс — открыть файл в репозитории → ✏️ → вставить новое содержимое → Commit, либо через терминал `git add . && git commit -m "..." && git push`)
3. Vercel автоматически пересобирает сайт за 30–60 секунд

## Стек

React + Vite. Деплой на Vercel. Данные рецептов/планов — в JS-файлах. Замены блюд в плане — в Supabase (отдельный проект, без логина, общий доступ по anon-ключу).

## Структура

```
src/
  App.jsx                 — всё приложение (UI, вкладки, навигация)
  lib/
    shoppingList.js       — расчёт списка покупок из плана недели
    supabase.js            — клиент Supabase
    useMealOverrides.js    — хук редактирования плана (замена блюд + сохранение)
  data/
    recipes.js             — все рецепты (КБЖУ, структурированные ингредиенты, шаги)
    weeks.js                — планы питания по неделям (ссылки на рецепты по дням)
supabase/
  schema.sql                — SQL для создания таблицы meal_overrides
```

## Как добавить новый рецепт

Открой `src/data/recipes.js`, скопируй один объект из массива `RECIPES`. Ингредиенты задаются через хелпер `ing(item, label, qty, unit, display, category, shelfLife)`:

- `id` — уникальный, на латинице, через дефис (на него ссылается план недели)
- `mealType` — `breakfast` / `lunch` / `pretrain` / `dinner` (slot `snack` в плане недели не ограничен типом — туда можно поставить рецепт любого из этих типов)
- `kcal`, `protein`, `fat`, `carbs` — числа, считаются по стандартным таблицам пищевой ценности на реалистичную порцию
- `activeTime`, `totalTime` — минуты
- `ingredients` — массив вызовов `ing(...)`:
  - `item` — нормализованный ключ продукта. **Важно:** если продукт уже встречается в других рецептах (например, лук, чеснок, куриная грудка), используй тот же `item`-ключ и ту же `unit` — иначе список покупок не сложит их вместе.
  - `label` — название для списка покупок
  - `qty` — число (или `null`, если количество не считается — «по вкусу», «щепотка»)
  - `unit` — единица (`г`, `мл`, `шт`, `зубчик`, `пучок`, `ч. л.`, `ст. л.` и т.п.)
  - `display` — как показать в самом рецепте (можно с «≈» и комментарием)
  - `category` — `protein` / `grains` / `dairy` / `produce` / `pantry` / `supplement`
  - `shelfLife` — `long` (закупаем сразу на неделю) / `fresh` (докупаем по ходу) / `skip` (не идёт в список, например вода)
- `steps` — массив строк (шаги приготовления)
- `look` — как выглядит готовое блюдо
- `tip` — совет тренера
- `link` (опционально) — `{ label, url }`, если нужна внешняя ссылка на источник рецепта

## Как добавить новую неделю

Открой `src/data/weeks.js`, скопируй один объект из массива `WEEKS` и измени:

- `id` — дата начала недели в формате `YYYY-MM-DD`
- `label` — как показывать диапазон, например `06.07 — 12.07`
- `days` — 7 дней, в каждом ссылки на `id` рецептов из `recipes.js`: `breakfast` / `lunch` / `pretrain` / `dinner` обязательны, `snack` (доп. перекус) — опционален, добавляй если нужно подвести калораж дня к цели
- Если в какой-то день нужно разовое блюдо вне каталога или ручная добавка к обычному рецепту (например «+100 г творога») — используй `*Custom` (текст на экране) и `*Extra` (объект или массив объектов в том же формате, что аргументы `ing(...)`, без `display` — он не нужен для списка покупок)

Список покупок строится автоматически — отдельно его прописывать не нужно.

Новая неделя автоматически появится в переключателе недель на вкладках «План недели» и «Покупки» (последняя добавленная неделя показывается по умолчанию).

## Редактирование плана прямо в интерфейсе

На вкладке «План недели» рядом с каждым блюдом есть кнопка ✏️ — открывает список блюд того же типа (для «доп. перекуса» — весь каталог, сгруппированный по типу) и позволяет заменить блюдо дня. Замена:

- сразу видна в плане (с пометкой «заменено»)
- учитывается в автоматическом расчёте списка покупок на вкладке «Покупки»
- сохраняется навсегда через Supabase — видна при следующем визите на любом устройстве (если Supabase подключен, см. шаг 2 выше)

Кнопка «Вернуть исходное блюдо плана» в этом же окне убирает замену и возвращает то, что было изначально прописано в `weeks.js`.

## Логика списка покупок

`src/lib/shoppingList.js` проходит по всем 7 дням недели → для каждого слота берёт либо замену из Supabase (если она есть), либо дефолтное блюдо из `weeks.js` → собирает ингредиенты (+ ручные `*Extra`, если блюдо не заменено) → суммирует одинаковые продукты (по `item`+`unit`) → раскладывает по категориям и по `shelfLife` (купить сразу на неделю / докупать свежим).

На вкладке «Покупки» есть переключатель порций (×1 / ×1.5 / ×2) — по умолчанию ×1, то есть расчёт идёт ровно по рецептам без запаса. Если готовишь не только на себя или хочешь запас — переключи множитель. У каждой группы продуктов есть чекбокс «отметить всё».

## Обновление после первого деплоя (короткая версия)

1. Внести правки в `src/data/recipes.js` или `src/data/weeks.js` (или попросить Claude это сделать)
2. Залить изменённые файлы на GitHub в репозиторий
3. Vercel пересобирает сайт автоматически
