# Дневник питания

Справочник: рецепты, план питания на неделю, список покупок, профиль с расчётом нормы КБЖУ. Без логина — статичный сайт, читать с телефона (в т.ч. как Telegram Mini App).

Список покупок считается **автоматически** из ингредиентов выбранных на неделю рецептов. Блюда в плане можно заменять прямо в интерфейсе, БАДы — редактировать, профиль — заполнить один раз. Всё это сохраняется навсегда через Supabase.

## Запуск с нуля

Ниже — полный путь от пустой папки до рабочего сайта по ссылке. Если что-то из этого уже есть (Node.js, GitHub-аккаунт, аккаунт Vercel, Supabase) — просто пропусти этот шаг.

### Шаг 0. Что понадобится

- Аккаунт на [GitHub](https://github.com) — бесплатный
- Аккаунт на [Vercel](https://vercel.com) — бесплатный, удобно входить через GitHub
- Аккаунт на [Supabase](https://supabase.com) — бесплатный, нужен для того, чтобы замены блюд, профиль и БАДы сохранялись навсегда. Без этого шага сайт работает полностью, просто эти данные сбросятся при перезагрузке страницы.
- Node.js на компьютере — нужен **только** если хочешь проверять сайт локально перед заливкой.

Скачать Node.js: [nodejs.org](https://nodejs.org) → версия LTS → установить как обычную программу.

### Шаг 1. Распаковать проект

Распакуй архив `nutrition-app.zip` в любую папку на компьютере, например на рабочий стол.

### Шаг 2. Завести проект в Supabase

Это отдельный, новый проект Supabase — никак не связанный с трекером тренировок.

1. Зайди на [supabase.com](https://supabase.com), войди (можно через GitHub)
2. **New Project** → дай название, например `nutrition-app` → выбери регион → задай пароль базы (можно сгенерировать и просто сохранить) → **Create new project**. Подождать минуту.
3. В левом меню → **SQL Editor** → **New query**
4. Открой файл `supabase/schema.sql` из архива, скопируй всё содержимое, вставь в редактор → **Run**. Это создаст три таблицы: замены блюд в плане, профиль и БАДы.
5. В левом меню → **Settings** (шестерёнка) → **API**
6. Скопируй два значения:
   - **Project URL** (выглядит как `https://xxxxx.supabase.co`)
   - **anon public** ключ (длинная строка под "Project API keys")

Эти два значения понадобятся в шаге 5 при подключении к Vercel.

### Шаг 3. (Опционально) Проверить локально перед заливкой

1. Открой терминал
2. ```
   cd путь/к/папке/nutrition-app
   ```
3. Скопируй `.env.example` в `.env` и впиши туда значения из Supabase (шаг 2)
4. ```
   npm install
   ```
5. ```
   npm run dev
   ```
6. Открой адрес из терминала (обычно `http://localhost:5173`)
7. `Ctrl + C` — остановить

### Шаг 4. Завести репозиторий на GitHub

1. [github.com/new](https://github.com/new) → название, например `nutrition-app` → Public/Private на вкус → **Create repository**
2. На странице нового репозитория → **«uploading an existing file»** → перетащи все файлы и папки из распакованного `nutrition-app` (включая `src` целиком, `supabase`, `package.json`, `index.html`, `vite.config.js`, `.gitignore`, `.env.example`, `README.md`) → **«Commit changes»**

`.env` (без `.example`) заливать не нужно — `.gitignore` его исключает.

Через терминал:
```
cd путь/к/папке/nutrition-app
git init
git add .
git commit -m "Первая версия"
git branch -M main
git remote add origin https://github.com/ТВОЙ-АККАУНТ/nutrition-app.git
git push -u origin main
```

### Шаг 5. Подключить к Vercel

1. [vercel.com](https://vercel.com) → войти через GitHub → **«Add New...» → «Project»** → найти репозиторий → **«Import»**
2. Framework Preset определится сам как **Vite**
3. Разверни **«Environment Variables»**, добавь:
   - `VITE_SUPABASE_URL` = Project URL
   - `VITE_SUPABASE_ANON_KEY` = anon public ключ
   - Отметь **Production** и **Preview** для каждой
4. **«Deploy»** → через 30–60 секунд получишь ссылку вида `nutrition-app-xxxx.vercel.app`

Если пропустил Supabase — сайт всё равно полностью работает, просто замены блюд/профиль/БАДы будут сбрасываться при перезагрузке. Подключить можно и позже: добавь переменные в Vercel и сделай **Redeploy**.

### Шаг 6. (Опционально) Закрепить домен

**Settings → Domains** на Vercel, либо переименуй проект в **Settings → General → Project Name**.

### Шаг 7. (Опционально) Telegram Mini App

1. [@BotFather](https://t.me/BotFather) в Telegram → `/newbot` или выбрать существующего
2. `/mybots` → бот → **Bot Settings → Menu Button → Configure Menu Button**
3. Вставь постоянный домен с Vercel, укажи название кнопки

### Как обновлять сайт после первого запуска

1. Меняешь файл (`src/data/recipes.js` или `src/data/weeks.js`) — сам или присылаешь мне, я меняю и отдаю обратно
2. Заливаешь изменённый файл на GitHub
3. Vercel пересобирает сайт автоматически за 30–60 секунд

## Стек

React + Vite. Деплой на Vercel. Данные рецептов/планов — в JS-файлах. Замены блюд, профиль и БАДы — в Supabase (отдельный проект, без логина, общий доступ по anon-ключу).

## Структура

```
src/
  App.jsx                 — всё приложение (UI, вкладки, навигация)
  lib/
    shoppingList.js       — расчёт списка покупок из плана недели
    macroCalculator.js    — формула Миффлина-Сан Жеора, расчёт целевых КБЖУ
    supabase.js           — клиент Supabase
    useMealOverrides.js   — хук редактирования плана (замена блюд + сохранение)
    useProfile.js         — хук профиля (рост/вес/возраст/пол/активность/цель)
    useSupplements.js     — хук редактируемого списка БАДов
  data/
    recipes.js            — все рецепты (КБЖУ, структурированные ингредиенты, шаги)
    weeks.js               — планы питания по неделям (ссылки на рецепты по дням)
supabase/
  schema.sql               — SQL для создания всех таблиц (meal_overrides, user_profile, supplements)
```

## Профиль и расчёт нормы КБЖУ

Кнопка-иконка профиля в шапке открывает форму: пол, возраст, рост, вес, уровень активности, цель (похудение / рекомпозиция / поддержание / набор массы). Расчёт идёт по формуле Миффлина-Сан Жеора:

- BMR (базовый обмен) = 10×вес + 6.25×рост − 5×возраст (+5 для мужчин, −161 для женщин)
- TDEE = BMR × коэффициент активности
- Целевые ккал = TDEE с поправкой на цель (−20% похудение, −10% рекомпозиция, 0% поддержание, +12% набор)
- Белок считается от веса тела (1.8–2.2 г/кг по цели), жир — 0.9 г/кг, углеводы — оставшиеся калории

Профиль сохраняется в Supabase, не нужно вводить заново при следующем визите.

## Как добавить новый рецепт

Открой `src/data/recipes.js`, скопируй один объект из массива `RECIPES`. Ингредиенты задаются через хелпер `ing(item, label, qty, unit, display, category, shelfLife)`:

- `id` — уникальный, на латинице, через дефис
- `mealType` — `breakfast` / `lunch` / `pretrain` / `dinner` (slot `snack` в плане недели не ограничен типом)
- `kcal`, `protein`, `fat`, `carbs` — числа, считаются по стандартным таблицам пищевой ценности на реалистичную порцию
- `activeTime`, `totalTime` — минуты
- `ingredients` — массив вызовов `ing(...)`:
  - `item` — нормализованный ключ продукта. Если продукт уже встречается в других рецептах, используй тот же `item`-ключ — иначе список покупок не сложит их вместе.
  - `label` — название для списка покупок
  - `qty` — число в граммах (или мл для жидкостей). Все ингредиенты, включая штучные (банан, яйцо, зубчик чеснока, пучок зелени), хранятся в граммах для точного сложения.
  - `unit` — всегда `г` или `мл`
  - `display` — как показать в самом рецепте, например `"120 г (≈1 шт)"` или `"3 г (≈1 зубчик)"`
  - `category` — `protein` / `grains` / `dairy` / `produce` / `pantry` / `supplement`
  - `shelfLife` — `long` (закупаем сразу на неделю) / `fresh` (докупаем по заходам) / `skip` (не идёт в список, например вода)
- `steps` — массив строк (шаги приготовления)
- `look` — как выглядит готовое блюдо
- `tip` — совет тренера
- `link` (опционально) — `{ label, url }`, внешняя ссылка на источник

Штучные продукты, для которых список покупок переводит граммы обратно в штуки (округляя вверх), перечислены в `PIECE_WEIGHTS` внутри `shoppingList.js` — если добавляешь новый штучный продукт, впиши туда его средний вес.

## Как добавить новую неделю

Открой `src/data/weeks.js`, скопируй один объект из массива `WEEKS`:

- `id` — дата начала недели, `YYYY-MM-DD`
- `label` — диапазон для отображения, например `06.07 — 12.07`
- `days` — 7 дней: `breakfast` / `lunch` / `pretrain` / `dinner` обязательны, `snack` — опционален
- `*Note` (например `breakfastNote: 'Новое блюдо недели'`) — текстовая пометка под названием блюда, показывается прямо в карточке дня. Значение `'Новое блюдо недели'` выводит золотой бейдж.
- Если нужна разовая ручная добавка к рецепту (например «+100 г творога») — `*Extra` (объект или массив в формате аргументов `ing(...)`)

## Редактирование плана прямо в интерфейсе

Кнопка ✏️ рядом с каждым блюдом на вкладке «План недели» открывает список блюд того же типа (для «доп. перекуса» — весь каталог, сгруппированный) и заменяет блюдо. Замена сразу видна (пометка «заменено»), учитывается в списке покупок и сохраняется в Supabase навсегда. «Вернуть исходное блюдо плана» — отменяет замену.

## БАДы

На вкладке «План недели» внизу — редактируемый список БАДов: название, порция, единица измерения, время приёма. Можно добавлять свои, редактировать и удалять существующие — нажми на строку. Сохраняется в Supabase.

## Логика списка покупок

`src/lib/shoppingList.js` собирает ингредиенты выбранных рецептов (с учётом замен из Supabase) → суммирует одинаковые продукты по `item`+`unit`. Продукты с `shelfLife: 'long'` идут в единый список на всю неделю. Продукты с `shelfLife: 'fresh'` разбиваются на **два захода закупки**:

- Заход 1 (купить в Вс/Пн) — на дни ПН–СР
- Заход 2 (купить в Ср) — на дни ЧТ–ВС

Так свежие продукты не лежат всю неделю и не портятся раньше времени.

Штучные продукты (авокадо, банан, помидор, лук, огурец, яйца, хлеб, болгарский перец, лаваш) показываются в списке покупок как целое количество штук, округлённое вверх — нельзя купить половину авокадо.

Переключатель порций (×1 / ×1.5 / ×2) — по умолчанию ×1 (точно по рецептам, без запаса). У каждой группы продуктов — чекбокс «отметить всё».
